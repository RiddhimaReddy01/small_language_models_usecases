================================================================================
THRESHOLD CALIBRATION PROCESS (Happens on VAL data)
================================================================================

BEFORE DEPLOYMENT:
  1. Use weights learned from TRAIN split
  2. Score all VAL samples: difficulty_score = P(SLM fails)
  3. For each possible threshold tau:
     - Route samples with score < tau to SLM
     - Route samples with score >= tau to LLM
     - Measure: Can SLM + LLM exceed capability threshold?
     - Measure: Does LLM error rate exceed risk limit?
  4. Select tau* that satisfies constraints

CONSTRAINTS APPLIED (typically):
  - Capability threshold (cap_min): LLM must succeed at rate > cap_min
  - Risk threshold (risk_max): Error rate must be < risk_max

EXPECTED RESULT:
  - High val success -> Low tau* (aggressive SLM routing)
  - Low val success -> High tau* (conservative LLM routing)
