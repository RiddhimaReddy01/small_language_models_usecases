4. The S³ Framework
S³ is a pre-deployment multicriteria decision-making framework that maps an enterprise workload to one of three deployment tiers before any inference infrastructure is provisioned. It consists of four components: a manager input protocol, a six-dimension rubric, a weighted sum formula with a normalization denominator, and a set of prescreening gate rules that enforce non-compensatory safety constraints. The framework is implemented in sddf/s3_framework.py.
4.1 Manager Input
A domain expert and a compliance reviewer jointly score the target workload using the anchored rubric in Table 2. Each dimension is assigned an integer score from 1 to 5. Scores are committed to the repository before any model inference is run; no revision is permitted after benchmarking. The scoring worksheet for all eight use cases is available in docs/s3_scoring_worksheet.md.

4.2 The S³ Formula
The S³ score is the weighted sum model normalized to the [1, 5] scale:
S³ = ( Σᵢ  scoreᵢ × wᵢ ) / ( Σᵢ  5 × wᵢ ) × 5
where scoreᵢ ∈ {1, 2, 3, 4, 5} is the expert score on dimension i and wᵢ is the dimension weight. The denominator Σᵢ 5×wᵢ equals 75 under the default weight vector, anchoring the score to the maximum achievable weighted sum. This ensures S³ values are directly comparable across workloads regardless of dimension count. The default weight vector is:
w = (TC=3,  OS=2,  SK=4,  DS=2,  LT=3,  VL=1)
The weight vector reflects a default enterprise risk profile in which stakes dominate, followed by task complexity and latency, with volume as the smallest tiebreaker. The constraint w_SK ≥ w_TC is enforced at runtime in compute_s3_score() to prevent any weight profile from prioritizing capability over safety. A lower S³ score indicates higher suitability for local SLM deployment; a higher score indicates the workload demands LLM capability or oversight.
4.3 Prescreening Gate Rules
Weighted sums allow high scores on some dimensions to compensate for low scores on others. In safety-sensitive settings this behavior is undesirable. Three non-compensatory gate rules are evaluated before the formula. Gate results are computed by prescreen_gate() in sddf/s3_framework.py and cannot be overridden by any weight perturbation.
Hard Rule 1 (SK = 5 → Disqualified). Any workload where SK = 5 is assigned LLM Only with mandatory human oversight, regardless of the formula score. Clinical triage (UC6, S³ = 4.27) triggers this rule because patient under-triage is irreversible and legally consequential.
Hard Rule 2 (TC = 5 and SK ≥ 4 → Disqualified). Expert-level generative tasks under significant stakes are disqualified autonomously. Financial report drafting (UC8, S³ = 3.07) triggers this rule: its formula score lies below τ1 = 3.2 and would be assigned Pure SLM by the formula alone, but TC = 5 combined with SK = 4 overrides the formula. The benchmark confirms the gate is load-bearing: the best SLM achieves only 89.7% parity (Llama-3.2-3B, 72.3% vs LLM 80.6%).
Flag Rule (SK ≥ 4 → minimum Hybrid). The formula may push higher but not lower than Hybrid. UC1 (S³ = 3.40) and UC7 (S³ = 3.20) trigger this rule. UC7 is the critical boundary case: S³ = 3.20 falls exactly on τ1; the Flag Rule is load-bearing since without it the rounding would depend on floating-point precision.
4.4 Tier Mapping
After gate evaluation, the formula score maps to one of three tiers through two calibrated thresholds (τ1 = 3.2, τ2 = 4.0), implemented in tier_from_s3():
Pure SLM     if  S³ ≤ 3.2      (and no gate disqualification)
Hybrid       if  3.2 < S³ ≤ 4.0   (or Flag Rule applies)
LLM Only     if  S³ > 4.0      (or Hard Rule 1 or 2 applies)

4.5 Worked Example: UC4 (Product Review Sentiment)
UC4 is a three-class sentiment classification task (POSITIVE, NEGATIVE, NEUTRAL) on publicly available product reviews. Dimension scores: TC = 1 (near-binary difficulty; POSITIVE and NEGATIVE carry strong lexical signals, NEUTRAL is the only ambiguous class), OS = 5 (single word from closed vocabulary), SK = 1 (wrong label causes analytics noise; no person harmed), DS = 1 (fully public data), LT = 3 (batch pipeline; P95 < 2 s acceptable), VL = 3 (tens of thousands per day).
Weighted sum:
(1×3) + (5×2) + (1×4) + (1×2) + (3×3) + (3×1)  =  3 + 10 + 4 + 2 + 9 + 3  =  31
S³ = (31 / 75) × 5 = 2.07
Gate check: SK = 1 → Pass (no rule fires). Tier: 2.07 ≤ 3.2 → Pure SLM. Benchmark confirms: Mistral-7B achieves 94.4% accuracy (96.7% LLM accuracy, 97.6% parity). Three additional SLMs exceed 90% parity. The gate-pass and Pure SLM assignment are correct. Note: the gold-set metadata records S³ = 2.1 (rounded), consistent with the computed value of 2.07.
