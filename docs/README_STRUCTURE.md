# Repository Structure

## data/
- `01_processed/`: persistent benchmark outputs, now containing the prior `benchmark_output`, `benchmark_output_fixed`, and `benchmark_output_fixed_all` runs.
- `02_sampling/`: placeholder for stratified sampling artifacts (per-bin definitions, sample manifests) generated by dataset-sampling stages.
- `03_complexity/`: holds the annotated difficulty scores, feature weights, and any derived task-complexity metadata pulled from SDDF.

This layout mirrors the redesigned pipeline: stage 1 archives raw benchmarks under `01_processed`, stage 2 records stratified sampling artifacts under `02_sampling`, stage 3 stores difficulty annotations + learned weights under `03_complexity`, and the routed SDDF math consumes those artifacts to produce capability/risk analysis and Pareto summaries.
