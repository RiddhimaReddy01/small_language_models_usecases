================================================================================
OPERATIONAL ZONE: DYNAMIC TAU BY TASK FAMILY
================================================================================

KEY INSIGHT:
  TAU is NOT a fixed single value
  TAU is a DYNAMIC RANGE (operational zone) that depends on task difficulty
  
  Operational Zone = [TAU_min, TAU_max]
                   = all valid difficulty thresholds where BOTH:
                     - capability >= cap_dyn
                     - risk <= risk_dyn

================================================================================
FORMULA: Why Hard Tasks Have Smaller Zones
================================================================================

Zone size ∝ (baseline_cap - cap_dyn)

EASY task:
  zone_size ∝ (0.99 - 0.70) = 0.29 (LARGE gap)
  Many difficulty levels fit between 0.70 and 0.99
  
MEDIUM task:
  zone_size ∝ (0.85 - 0.55) = 0.30 (MEDIUM gap)
  Some difficulty levels fit between 0.55 and 0.85
  
HARD task:
  zone_size ∝ (0.75 - 0.70) = 0.05 (SMALL gap)
  Few difficulty levels fit between 0.70 and 0.75
  
IMPOSSIBLE task:
  zone_size ∝ (0.65 - 0.65) = 0.00 (NO gap)
  No difficulty levels between 0.65 and 0.65 (empty)

================================================================================
DYNAMIC TAU SELECTION STRATEGY
================================================================================

For each task family:

Step 1: Compute operational zone F (all valid difficulty levels)

Step 2: Choose routing strategy based on zone size:

  IF |F| = 0 (empty zone):
    TAU = None  (no routing possible)
    "Always use baseline"
  
  ELIF |F| >= 3 (wide zone - easy task):
    TAU = max(F)  (aggressive routing)
    "Route many queries, save costs"
  
  ELIF |F| = 2 (medium zone):
    TAU = mean(F)  (balanced routing)
    "Route some queries, balance cost/quality"
  
  ELIF |F| = 1 (narrow zone - hard task):
    TAU = min(F)  (conservative routing)
    "Route only essential queries"

================================================================================
SUMMARY: TAU IS DYNAMIC & TASK-FAMILY SPECIFIC
================================================================================

Traditional approach:
  "Find THE TAU for each task"
  TAU is a single fixed number
  Same TAU for all tasks

Correct approach:
  "Find THE OPERATIONAL ZONE for each task"
  Operational zone is a RANGE [TAU_min, TAU_max]
  Zone size DEPENDS on task difficulty
  
  Easy tasks:   WIDE zones, aggressive routing possible
  Hard tasks:   NARROW zones, conservative routing only
  Impossible:   EMPTY zones, no routing possible

Decision:
  Based on business goals, choose a TAU from within the zone
  Conservative: pick TAU_min (minimize risk)
  Aggressive: pick TAU_max (maximize savings)
  Balanced: pick (TAU_min + TAU_max) / 2
