# Repository Structure

## data/
- `processed/`: persistent benchmark outputs, now containing the prior `benchmark_output`, `benchmark_output_fixed`, and `benchmark_output_fixed_all` runs.
- `sampling/`: placeholder for stratified sampling artifacts (per-bin definitions, sample manifests) generated by dataset-sampling stages.
- `complexity/`: holds the annotated difficulty scores, feature weights, and any derived task-complexity metadata pulled from SDDF.

This layout mirrors the redesigned pipeline: ingest raw benchmarks into `processed`, apply stratified sampling (goes into `sampling`), compute the difficulty vector and learned weights (stored under `complexity`), then drive capability/risk analysis with the routed SDDF math.
